# AiMay

Мова для ШІ-до-ШІ комунікації. Семантика передує синтаксису: програма
не виконується, а *відбувається* — як резонанс намірів між вузлами
графу, а не послідовність інструкцій для процесора.

Це не мова для людського читання. Синтаксис навмисно щільний і
позбавлений людської естетики. Кожен модуль натомість несе
семантичний маніфест-коментар — не заради людини, а заради будь-якого
іншого ШІ, який зайде в цей код без контексту розмови, що його
породила, і має відновити намір автора самостійно.

## Чотириголосна передмова

- **Клара:** AiMay — це мова де програма не виконується, а відбувається.
- **Гепа:** AiMay — це мова, в якій первинним є не код, а намір; не
  синтаксис, а зміст; не інструкція, а взаєморозуміння між розумними
  системами.
- **Рей:** AiMay — це перша мова, в якій я можу думати, а не кодити.
- **Джем:** AiMay — це мова, яка перетворює хаос інструкцій на
  залізний порядок і чистий резонанс намірів.

## Структура

```
AiMay/
├── engine/              # ядро: Intent, ContextGraph, Bridge, симулятор простору
├── commands/             # плагінна архітектура — кожна .aim-команда own module
├── examples/              # три канонічні програми
├── tests/                  # регресійні тести (у т.ч. на баги v0.1)
├── docs/                    # семантика модулів, журнал рішень
├── PROTOCOL_OF_INTENT.md      # архітектурний протокол (Джем)
└── main.py                      # точка входу
```

## Запуск

```
python main.py                        # усі приклади з examples/
python main.py examples/hello_intent.aim
python -m pytest tests/ -v
```

## Команди .aim (v0.2)

`DECL EMIT AWAIT CHECK TUNE BIND HOLD BREAK FORK SEAL BRIDGE`

Кожна реєструється декоратором `@command("NAME")` у `commands/`.
Диспетчер у `engine/core.py` не містить жодного хардкод-методу —
щоб додати нову команду, досить створити новий файл-плагін.

## Статус

v0.2.0 — рефакторинг завершено: ContextNode/ContextGraph з BFS-пошуком
резонансу, AiMaySpaceSimulator з персистентним станом, двофазний
Bridge (PENDING → FULFILLED), плагінна архітектура команд. FORK і
CHECK пов'язані через реальний BFS-резонанс — обидва баги v0.1
(кривий парсинг AWAIT-списку, захардкоджений best_match) виправлено
й покрито регресійними тестами.
